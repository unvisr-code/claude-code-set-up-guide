# Claude Code 설치 가이드 단일 페이지 PRD

## 0. 개요

* 문서명: Claude Code 설치 가이드 단일 페이지 PRD
* 목적: 경기 기후 바이브코딩 해커톤 참가자가 **단일 웹페이지**를 보고 Windows 또는 macOS에서 Claude Code CLI를 스스로 설치할 수 있게 한다.
* 페이지 타입: 랜딩 형태의 **단일 페이지(single page)** 웹사이트
* 로그인/계정 연동: 이 페이지에서는 다루지 않음 (별도 페이지/가이드)

---

## 1. 목표 & 성공 기준

### 1.1 목표

* 비개발자/초보자도 이 페이지의 안내만 보고

  * 자신의 운영체제를 선택하고,
  * 순서대로 명령어를 따라 입력하면,
  * **Claude Code CLI 설치 및 버전 확인까지 완료**할 수 있도록 한다.

### 1.2 성공 기준 (정성/정량)

* 사전 지식이 거의 없는 참가자 3명 이상에게 테스트했을 때, **추가 설명 없이 80% 이상이 설치 완료**.
* 페이지 이탈 없이 평균 1회 방문으로 설치 완료(로그 수집 시).

---

## 2. 타깃 유저 & 사용 시나리오

### 2.1 타깃 유저

* 경기 기후 바이브코딩 해커톤 참가자
* Windows 노트북 또는 macOS 노트북을 사용하는 비개발자/초보자
* 터미널 개념은 낯설지만, 텍스트를 보고 그대로 따라 하는 것은 가능

### 2.2 사용 시나리오

1. 해커톤 안내 메일/노션/카톡에서 **설치 가이드 링크**를 클릭한다.
2. 단일 페이지가 열리고, 상단에서 행사명과 간단한 설명을 본다.
3. 중단의 **운영체제 선택 토글(Windows/macOS)** 에서 자신의 OS를 선택한다.
4. 선택된 OS의 **1단계부터 순서대로** 읽고,

   * 코드블록 복사 버튼을 눌러
   * PowerShell/Ubuntu 터미널/맥 터미널에 붙여넣고 실행한다.
5. 마지막 단계의 `claude --version` 명령까지 실행해 **버전이 출력되면 설치 성공**임을 확인한다.
6. 페이지 하단의 안내에 따라 "다음: 로그인 가이드" 링크로 이동한다.

## 3. 페이지 구조 (IA & Layout)

### 3.1 전체 레이아웃

* 단일 컬럼 구조, 상단에서 하단으로 자연스럽게 내려가는 스크롤형 페이지.
* 주요 섹션 순서

  1. 히어로 섹션 (행사 브랜딩 + 페이지 목적 설명)
  2. 운영체제 선택 섹션 (Windows / macOS 토글)
  3. OS별 설치 단계 섹션 (Windows 또는 macOS 중 하나만 표시)
  4. 설치 완료/다음 단계 안내 섹션

### 3.2 히어로 섹션

* 배경: 제공된 현수막을 참고한 **다크 네이비 → 블랙 그라디언트**.
* 요소

  * 상단 좌측: 작은 태그 텍스트

    * `경기 기후 바이브코딩 해커톤`
  * 메인 타이틀 (큰 흰색 텍스트)

    * `Claude Code 설치 가이드`
  * 서브텍스트

    * `이 페이지에서는 Windows와 macOS에서 Claude Code CLI를 설치하는 방법을 안내합니다.`
    * `로그인·계정 연동은 다음 단계에서 진행합니다.`
  * 행사 정보 (작은 텍스트)

    * `운영: Second Team (시소)  |  주최: 경기도`
* 전체적인 톤은 제공된 배너와 유사하게: 어두운 배경 + 굵은 타이틀 + 좌우 여백 넉넉하게.

### 3.3 운영체제 선택 섹션

* 히어로 바로 아래에 위치.
* 내용

  * 섹션 타이틀: `운영체제를 선택하세요`
  * 설명: `사용 중인 컴퓨터에 맞는 탭을 선택하면, 해당 OS 전용 설치 방법이 아래에 표시됩니다.`
* UI

  * 두 개의 탭 또는 세그먼트 버튼 형태

    * `macOS` (애플 모양 아이콘)
    * `Windows` (모니터 또는 Windows 로고 느낌 아이콘)
  * 클릭 시 해당 OS 섹션만 아래에 표시, 다른 OS 섹션은 완전히 숨김.
  * 기본 선택: (선택 사항) 브라우저 User-Agent 기반으로 자동 감지. 감지 실패 시 `Windows`를 기본값으로.

### 3.4 OS별 설치 단계 섹션 공통 규칙

* 각 OS는 **단계 카드 리스트**로 구성 (1단계 ~ N단계).
* 각 단계 카드 구성

  * 번호 + 제목 (예: `1. WSL 설치하기`)
  * 1~3줄 요약 설명
  * 구체적인 행동 순서 (번호 매긴 리스트)
  * 실행해야 할 명령어가 있을 경우 **코드 블록 + 복사 버튼** 포함
  * 필요하면 주의/정보 카드(노란색 또는 파란색 경고 박스)
* 모든 문장은 컴퓨터 초보 입장에서 작성 (반말/존댓말 스타일은 서비스 톤앤매너에 맞춰 존댓말로 가정).

## 4. Windows 설치 가이드 섹션 상세

### 4.0 사전 안내 카드

* 카드 내용 예시:

  * 제목: `Windows에서 설치하기 전에`
  * 텍스트:

    * `이 가이드는 Windows 10 또는 11, 64비트 기준입니다.`
    * `설치 중에는 PC를 한두 번 재부팅해야 할 수 있습니다.`
    * `아래 단계를 **1번부터 순서대로** 따라 해 주세요.`

### 4.1 1단계 – WSL 설치하기

* 제목: `1. WSL(Windows Subsystem for Linux) 설치`
* 설명 문구:

  * `WSL은 윈도우 안에서 리눅스 터미널을 사용할 수 있게 해주는 기능입니다.`
  * `먼저 WSL을 설치해 리눅스 환경(Ubuntu)을 준비합니다.`
* 행동 순서:

  1. 화면 왼쪽 아래 **검색창**에 `PowerShell`을 입력합니다.
  2. 검색 결과에서 `Windows PowerShell`을 **마우스 오른쪽 버튼**으로 클릭하고, `관리자 권한으로 실행`을 선택합니다.
  3. 파란색 창이 열리면, 아래 명령어를 그대로 복사해 붙여넣고 **Enter** 키를 누릅니다.
* 코드 블록:

  ```bash
  wsl --install
  ```
* 추가 안내:

  * `명령 실행 후 설치가 진행되며, 안내에 따라 컴퓨터를 **재부팅**해 주세요.`
  * `재부팅 후 첫 실행 시 Ubuntu 설치 화면이 자동으로 나타날 수 있습니다.`

### 4.2 2단계 – Ubuntu 설치 및 처음 실행

* 제목: `2. Ubuntu 설치 및 계정 만들기`
* 설명:

  * `WSL 위에서 돌아가는 리눅스 배포판인 Ubuntu를 설치합니다.`
* 행동 순서:

  1. 재부팅 후, 만약 자동으로 **Ubuntu 창**이 뜨면 그대로 기다립니다. 자동으로 뜨지 않으면 **시작 메뉴**에서 `Ubuntu`를 검색해 실행합니다.
  2. `Installing, this may take a few minutes...` 라는 문구가 보이면 설치가 진행 중이니 기다립니다.
  3. 설치가 끝나면 `Enter new UNIX username:` 이라는 문장이 나타납니다.

     * 영문 소문자로 원하는 이름을 입력합니다. (예: `jun`, `shop2world`)
  4. 비밀번호를 두 번 입력합니다.

     * 화면에 글자가 보이지 않아도 정상입니다.
* 주의 카드:

  * `⚠️ 앞으로 나오는 모든 명령어는 이 Ubuntu 창(검정 또는 보라색 터미널)에서 입력해야 합니다.`

### 4.3 3단계 – Node.js와 npm 설치하기

* 제목: `3. Node.js와 npm 설치`
* 설명:

  * `Claude Code는 Node.js 기반 도구이므로, 먼저 Node.js와 패키지 관리자 npm을 설치합니다.`
* 행동 순서:

  1. Ubuntu 터미널에서 아래 명령어를 순서대로 한 줄씩 입력하고, 각 줄마다 Enter를 누릅니다.
* 코드 블록 1:

  ```bash
  sudo apt update
  ```
* 코드 블록 2:

  ```bash
  sudo apt install -y nodejs npm
  ```
* 설치 확인 코드 블록:

  ```bash
  node -v
  npm -v
  ```
* 확인 설명:

  * `각 명령을 실행했을 때 'v18.0.0'처럼 v로 시작하는 숫자가 나오면 정상 설치된 것입니다.`

### 4.4 4단계 – Claude Code CLI 설치하기

* 제목: `4. Claude Code 설치`
* 설명:

  * `이제 Claude Code CLI를 전역(global)으로 설치합니다.`
* 행동 순서:

  1. Ubuntu 터미널에서 아래 명령어를 입력하고 Enter를 누릅니다.
* 코드 블록:

  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
* 주의 카드:

  * 텍스트: `설치 중에 'EACCES' 또는 'permission denied' 같은 권한 오류가 나면, 아래처럼 앞에 sudo를 붙여 다시 실행해 주세요.`
* 대체 코드 블록:

  ```bash
  sudo npm install -g @anthropic-ai/claude-code
  ```

### 4.5 5단계 – 설치 확인하기

* 제목: `5. 설치가 잘 되었는지 확인`
* 설명:

  * `마지막으로 Claude Code가 제대로 설치되었는지 확인합니다.`
* 코드 블록:

  ```bash
  claude --version
  ```
* 성공 안내 카드 (초록색):

  * `버전 정보(예: claude 1.2.3)가 표시되면, Windows(Ubuntu)에서 Claude Code 설치가 완료된 것입니다! 🎉`
* 에러 안내 텍스트:

  * `만약 'command not found: claude' 같은 메시지가 나온다면, 설치가 제대로 되지 않은 것입니다. 이 화면을 캡처해 운영진/멘토에게 공유해 주세요.`

## 5. macOS 설치 가이드 섹션 상세

### 5.0 사전 안내 카드

* 카드 내용 예시:

  * 제목: `macOS에서 설치하기 전에`
  * 텍스트:

    * `이 가이드는 macOS 12 Monterey 이상을 기준으로 작성되었습니다. 그 이하 버전에서도 대부분 동일하게 동작하지만, 화면이 조금 다를 수 있습니다.`
    * `아래 단계를 **1번부터 순서대로** 따라 해 주세요.`

### 5.1 1단계 – 터미널 열기 & Homebrew 설치

* 제목: `1. 터미널 열기 & Homebrew 설치`
* 설명:

  * `Homebrew는 macOS용 프로그램 설치 도구입니다. Node.js를 쉽게 설치하기 위해 먼저 Homebrew를 설치합니다.`
* 행동 순서:

  1. 키보드에서 `⌘(Command) + Space` 를 눌러 **Spotlight 검색**을 엽니다.
  2. `터미널` 또는 `Terminal`을 입력하고, 나오는 **터미널 앱**을 실행합니다.
  3. 터미널 창이 열리면, 아래 명령어 전체를 복사해 붙여넣고 Enter를 누릅니다.
* 코드 블록:

  ```bash
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  ```
* 안내 텍스트:

  * `설치 과정에서 암호를 입력하라는 문구가 나오면, 사용 중인 맥의 로그인 비밀번호를 입력하고 Enter를 눌러 주세요.`
  * `설치에는 몇 분 정도 걸릴 수 있습니다.`
  * `설치가 끝난 후에는 터미널을 한 번 닫았다가 다시 열어 주세요.`

### 5.2 2단계 – Node.js와 npm 설치하기

* 제목: `2. Node.js 설치`
* 설명:

  * `Homebrew로 Node.js를 설치합니다.`
* 행동 순서:

  1. 새 터미널 창에서 아래 명령어를 입력하고 Enter를 누릅니다.
* 코드 블록:

  ```bash
  brew install node
  ```
* 설치 확인 코드 블록:

  ```bash
  node -v
  npm -v
  ```
* 설명:

  * `각 명령을 실행했을 때 'v18.0.0'처럼 v로 시작하는 숫자가 나오면 정상 설치된 것입니다.`

### 5.3 3단계 – Claude Code CLI 설치하기

* 제목: `3. Claude Code 설치`
* 설명:

  * `npm을 사용해 Claude Code CLI를 전역으로 설치합니다.`
* 행동 순서:

  1. 터미널에서 아래 명령어를 입력하고 Enter를 누릅니다.
* 코드 블록:

  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
* (선택) 권한 오류 시 안내 카드:

  * 텍스트: `권한 관련 오류가 나면, 아래 명령어로 다시 시도해 주세요.`
* 대체 코드 블록 (필요 시):

  ```bash
  sudo npm install -g @anthropic-ai/claude-code
  ```

### 5.4 4단계 – 설치 확인하기

* 제목: `4. 설치가 잘 되었는지 확인`
* 설명:

  * `아래 명령어를 실행해 Claude Code 버전이 정상적으로 표시되는지 확인합니다.`
* 코드 블록:

  ```bash
  claude --version
  ```
* 성공 카드:

  * `버전 정보가 표시되면, macOS에서 Claude Code 설치가 완료된 것입니다! 🎉`
* 에러 안내 텍스트:

  * `만약 'command not found: claude' 같은 메시지가 나온다면, 설치 경로 설정이 필요할 수 있습니다. 이 화면을 캡처해 운영진/멘토에게 공유해 주세요.`

## 6. 비기능 요구사항

1. **반응형 디자인**

   * 모바일, 태블릿, 데스크톱에서 모두 사용 가능해야 함.
   * 코드 블록은 모바일에서 가로 스크롤 가능하도록 처리.

2. **브랜딩**

   * 배경/컬러/타이포는 `경기 기후 바이브코딩 해커톤` 메인 배너를 기준으로 한다.
   * 다크 네이비 → 블랙 그라디언트 배경 + 화이트 타이틀 + 행사 정보 텍스트.

3. **UX 라이팅**

   * 모두 존댓말 사용.
   * "지금 해야 할 일"이 카드 상단 1~2줄에 요약되어 있어야 함.
   * 전문 용어 사용 시 괄호로 의미를 함께 표기 (예: `터미널(명령어를 입력하는 창)`).

4. **코드 복사 기능**

   * 모든 코드 블록 오른쪽 상단에 `복사` 아이콘 버튼.
   * 클릭 시 해당 코드 전체가 클립보드에 복사되고, `복사되었습니다` 토스트 메시지 표시.

## 8. 완료 정의 (Definition of Done)

* [ ] 단일 페이지에서 Windows / macOS 탭 전환이 정상 동작한다.
* [ ] 각 OS에 대해 0~5단계(또는 0~4단계)의 카드가 PRD 내용대로 구현되어 있다.
* [ ] 모든 코드 블록 복사 버튼이 정상 동작하며, 토스트 메시지가 표시된다.
* [ ] 모바일/데스크톱에서 레이아웃이 깨지지 않는다.
* [ ] 실제 참가자 테스트에서, 추가 설명 없이 Windows/macOS 각각 최소 1명 이상이 설치를 끝까지 완료했다.
